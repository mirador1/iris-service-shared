# Production Readiness Checklist — Mirador project family

> Cross-cutting reference. Each checklist item maps to **what's in the
> repo today** vs **what would be needed for an actual production rollout**.
> Useful for hiring conversations ("show me the production-readiness
> story") and as a self-audit before any real deploy.

## How to read

| Symbol | Meaning |
|---|---|
| ✅ | Done in the demo, exercised in CI |
| 🟡 | Partial : present but with caveats / informational gates |
| 🔴 | Not done — would be a real blocker for prod |
| ⚪ | Out of scope for this portfolio demo |

## Reliability

| Item | Status | Where / Why |
|---|---|---|
| **SLO defined for every customer-facing API** | ✅ | 3 SLOs per service via Sloth (availability 99% / latency p99 < 500ms / enrichment 99.5%). | **Runbook for every alert** | ✅ | [java runbooks](https://gitlab.com/mirador1/mirador-service-java/-/tree/main/docs/runbooks) + [python runbooks](https://gitlab.com/mirador1/mirador-service-python/-/tree/main/docs/runbooks) |
| **Health probes (liveness + readiness)** | ✅ | Spring Boot Actuator + FastAPI `/actuator/health/{liveness,readiness}` |
| **Graceful degradation on downstream failure** | ✅ | Resilience4j (Java) + tenacity (Python) on all external calls — retry + circuit-breaker + fallback |
| **Auto-rollback on canary failure** | ✅ | Argo Rollouts canary with prometheus analysis template |
| **Chaos testing** | 🟡 | Chaos Mesh scenarios + diagnostic endpoints (`/customers/diagnostic/*`) ; not run in CI |
| **RTO + RPO measured** | 🔴 | Targets in SLA doc but not actively measured ; would need a periodic chaos-driven measurement |

## What this list demonstrates

Every ✅ item maps to a **specific file or commit** in the repos — not just
intent. The 🟡 items have honest caveats. The 🔴 items show what would
actually need to happen for a real production rollout — and we don't
pretend they're done.

That honesty IS the recruiter signal : a portfolio that says "we ship
SLOs" but can't show the YAML is theatre. A portfolio that says
"we don't have mTLS yet, here's why we'd add Linkerd if we did" is
production-grade thinking.
